#### Time Format

The time string in JSON should be like this:

```Java
import java.time.Instant;
Instant.now().toString();
```

> 2017-01-23T12:34:56.123456789Z

Recover from string

```Java
Instant instant = Instant.parse( "2017-01-23T12:34:56.123456789Z" );
```

####  1. Create a Task

**Description:**
create a collection task or annotation task.  
TODO: XML standard (URL of pictures,task labels)

**Request:**

- URI

  ```http
  POST /api/tasks/
  ```


- Body
id 由后端生成
  ```Json
  {
      "data": {

          "formatter": {
            "tags":[
                {
                  "name":"tag1",
                  "description":"some description about tag1",
                  "attributes":[
                    {
                      "name":"attr1",
                      "description":"some description about attr1",
                      "values":["option1","option2"]
                    }
                  ]
                }
            ]

          },
          "title": "xxx",
          "start_time": "xxxx",
          "deadline": "xxx",
          "description": "xxx",
          "type":0
      }
  }
  ```

  type:  
  0: 标注
  1: 采集  


  formatter:
  1. APP对输入的属性取值(Json中的values字段)**不做类型区分**，一律存成String。
  2. 属性的values字段设置为空表示可以任意取值。否则标注用户只能从中选择(一个或者多个)。
  3. 由于标注方式太灵活，每个标签和属性均需要提供description字段用于指导用户如何进行标注。

**Response:**

- Status Code:  201 Created

- Body

  EMPTY

**Errors:**

- Permission denied

- Status Code: 401
- Body
```JSON
{
    "errors": {
        "title": "Please login",
        "detail": "Permission denied",
        "status": 401
    }
}
```

- File not found

- Status Code: 404

- Body

```Json
{
    "errors": {
        "title": "File not found",
        "detail": "picture url not exists",
        "status": 404
    }
}
```

#### 3. Get tasks id multiple
Init task window.
初始化界面时可能会需要请求任务信息，后端返回任务id，需要完善。

**Request:**
- URI
```HTTP
GET /api/tasks?limit=10/
```

**Response:**
- Status code: 200
- Body

```json
{
  "meta":{
  "result_set":{
    "total":100,
    "count":10,
    "next":"...&offset=10"
  }
  },
  "data":{
    "tasks":[
      {
          "id": 1,
          "name": "xxx",
          "description": "xxx",
          "start_time": "1234567890",
          "type": 0,
          "size": 100,
          "data_path":"xxx",
          "creator":0,
          "progress":20,
          "pictures":["url1","url2"],
          "deadline": "xxx",
          "formatter":{}
      }
    ]

  }
}
```
id： task_id
pictures: 提供最多10张此任务的图片方便前端预览
pictures 不是tasks 表中的列

Also support /api/tasks?task_id={task_id}
meta 如果"offset=10"中加"prev":"...&offset=0"
见 https://developer.digitalchalk.com/document/rest-api-v5/limit-and-offset/

**Description:**

- URI

  ```http

  ```

      }
  }

#### 6. Apply  a task

**Description:**
申请一个任务，后端发放一部分数据
**Request:**
- URI
```http
POST /api/tasks/apply/
```
- Body
```JSON
{
  "data":{
    "task_id":1,
    "user_id":0
  }
}
```
user_id : 申请者id

- Response
  size: 返回任务的大小（需要标注或者采集多少图片）
  pictures: 需要标注的图片
  标注任务
  ```json
  {
    "data":{
      "commit_id":0,
      "size":2,
      "task_id":1,
      "type":0,
      "pictures":[
        "picture_url1",
        "picture_ur2",
      ]
    }

  }
  ```

  采集任务

  ```json
  {

    "data":{
      "size":2,
      "task_id":1,
      "type":1
    }

  }
  ```

### Commits

####  1. Upload a commit

##### 1 标注任务

**Description:**
完成标注任务并且上传

**Request:**

- URI

  ```http
  POST /api/commits/
  ```


- Body

  标注任务
  ```Json
    {   
        "data": {
            "commit_id":0,
            "task_type":0,
            "task_id": 1,
            "user_id": 1,
            "size":1,
            "result":[
                {
                   "picture_url": "URL",
            		   "annotation_json": "xxx"
                }
            ]
        }
    }
  ```  
  xxx:
  ```json
  {
      "tags": [
          {
              "name": "tag1",
              "position": [
                  [0, 0],
                  [100, 100]
              ],
              "attributes": [
                  {
                      "name": "attr1",
                      "values": ["option1", "option2"]
                  }
              ]
          }
      ]
  }
  ```

1. position字段指定标签的位置。
   - 框图标注：提供左上角，右下角坐标。
   - 点标注(Optional): 提供所有点坐标。
2. APP对用户的输入字段**不做检查**。将处理过程交给任务发布方或者管理员。


**Response:**

- Status Code:  201 CREATED

- Body

  EMPTY

**Errors:**
- Permission denied
- Status Code: 401
- Body
```JSON
{
    "errors": {
        "title": "Please login",
        "detail": "Permission denied",
        "status": 401
    }
}
```

####  2. Get User Commits

**Description:**

**Request:**

- URI
  最后没有/
  ```http
  GET /api/commits?user={user_id}&task={task_id}&pageNum=0&pageSize=10
  ```


- Body

  EMPTY


**Response:**

- Status Code: 200 OK

- Body

  ```Json
  {   
    "meta":{

       "result_set": {
               "count": 3,
               "offset": 0,
               "limit": 3,
               "total": 77,
               "next":"/api/commits?user={user_id}&task={task_id}}&pageNum=1&pageSize=10"
           }
       },

      "data": {
        "commits":[
          {
          "commit_id": 0,
          "task_id": 2,
          "user_id": 1,
          "result": [
              {
                	 "picture_url": "url",
          		     "annotation_json_rul": "url"  
              }
                  ]
          }

      ]
      }
  }
  ```
  user_id : 提交者id

  备注：
  pageNum: 请求页编号
  pageSize: 请求页大小为多少
  举例：
  请求数据共有21个，前端一次需要5个，想请求6，7，。。10.则pageNum=1,pageSize=5
**Errors:**

- Commits not found

- Status Code:404

- Body

  ```Json
  {
      "errors": {
          "title": "You haven't take this task",
          "detail": "can not find commits of this task",
          "status": 404
      }
  }
  ```
